## v0.23.0

### English

#### Highlights

- Added a guided server setup path for Codex, Claude Code, and Antigravity CLI.
- Added full-backup migration from a local Clipal installation to a server.

#### What Changed

- `clipal init` detects or installs selected CLI tools, starts the Clipal user service, and applies supported user-level CLI takeover.
- `clipal init --import <backup>` restores Clipal data before server setup, then applies takeover for the server user.
- Global Settings now separately control remote proxy and remote Web UI access. Both are disabled by default.
- The System Status page includes a server setup checklist for service, Provider, and CLI takeover readiness.

#### Release Notes

- Existing `clipal.data/v1` backups remain compatible. Deployment-specific remote-access settings are intentionally not restored from backups.
- Antigravity CLI installation is supported; automatic Antigravity CLI takeover remains pending its official configuration contract.
- Remote Web UI access has no built-in authentication. Protect a deliberately exposed instance with a firewall, VPN, or reverse proxy.

### 中文

#### 亮点

- 新增面向 Codex、Claude Code 和 Antigravity CLI 的服务器初始化路径。
- 支持将本机 Clipal 的完整备份迁移到服务器。

#### 变更内容

- `clipal init` 会检测或安装所选 CLI、启动 Clipal 用户服务，并为支持的 CLI 应用用户级接管。
- `clipal init --import <backup>` 会在服务器初始化前恢复 Clipal 数据，随后为服务器用户重新应用接管。
- 全局设置新增远程代理与远程 Web UI 的独立开关，默认均为关闭。
- 系统状态页新增服务、Provider 和 CLI 接管就绪状态的服务器初始化检查清单。

#### 发布说明

- 现有 `clipal.data/v1` 备份保持兼容；部署环境的远程访问开关不会从备份恢复。
- 已支持安装 Antigravity CLI；其自动接管仍等待官方配置契约确认。
- 远程 Web UI 没有内置认证。主动开放实例时，请使用防火墙、VPN 或反向代理保护。
