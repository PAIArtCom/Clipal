## v0.18.1

### English

#### Highlights

- Synced Claude Code and Codex OAuth request wrapping with current CLI traffic.
- Added proactive OAuth token maintenance so stored OAuth credentials refresh before expiry.
- Hardened live Claude OAuth smoke testing to avoid invalidating real refresh tokens.

#### What Changed

- Fixed: Claude OAuth wrapping now targets Claude Code `2.1.161`, including the current `sdk-cli` user agent, billing entrypoint, billing version fingerprint, and Claude Agent SDK system baseline.
- Fixed: Claude `thinking` requests now include the `thinking-token-count-2026-05-13` beta only when the request body actually uses `thinking`; the removed `redact-thinking-2026-02-12` beta is no longer sent.
- Fixed: Synthetic Claude OAuth requests no longer forward stale non-CLI `X-App-Name`, `X-App-Ver`, `X-Client-App`, or unrelated inbound beta fingerprints.
- Improved: Codex OAuth wrapping now targets Codex `0.136.0` and forces the Codex CLI `User-Agent`, `Originator`, and `Version` headers for non-CLI callers.
- Fixed: Codex compact requests now always emit compaction turn metadata, even when the inbound request includes stale ordinary turn metadata.
- Improved: OAuth credentials are scanned every five minutes and refreshed at the earlier of 80% of token lifetime or the configured expiry skew.
- Fixed: Concurrent refreshes for the same OAuth credential now coalesce across HTTP client paths, reducing refresh-token rotation races.
- Fixed: The background OAuth refresh maintainer starts only after the proxy listener is bound and can be cancelled during shutdown.
- Improved: The Claude live OAuth smoke script strips refresh tokens from its temporary credential copy, caps `Retry-After`, avoids retaining sensitive temp data by default, and handles emoji/non-BMP prompts in billing fingerprints.

#### Release Notes

- No config migration is required.
- Existing OAuth credentials remain compatible. Credentials with valid refresh tokens will be maintained automatically once Clipal starts.
- The Claude live smoke script now refuses to run with an expired access token after stripping the refresh token from its temporary copy; reauthorize or refresh the source credential first, then rerun the smoke.

### 中文

#### 亮点

- 将 Claude Code 与 Codex OAuth 请求包装对齐到当前 CLI 流量形态。
- 新增 OAuth token 主动维护，在过期前自动刷新已保存凭据。
- 加固 Claude OAuth live smoke，避免测试过程让真实 refresh token 失效。

#### 变更内容

- 修复：Claude OAuth 包装对齐 Claude Code `2.1.161`，包括当前 `sdk-cli` user agent、billing entrypoint、billing version fingerprint 和 Claude Agent SDK system baseline。
- 修复：Claude `thinking` 请求只在 body 实际启用 `thinking` 时发送 `thinking-token-count-2026-05-13` beta；已移除的 `redact-thinking-2026-02-12` beta 不再发送。
- 修复：合成的 Claude OAuth 请求不再透传非 CLI 客户端带入的 stale `X-App-Name`、`X-App-Ver`、`X-Client-App` 或无关 beta 指纹。
- 改进：Codex OAuth 包装对齐 Codex `0.136.0`，并为非 CLI 调用强制使用 Codex CLI 的 `User-Agent`、`Originator` 和 `Version` header。
- 修复：Codex compact 请求现在总会生成 compaction turn metadata，即使入站请求带了旧的普通 turn metadata。
- 改进：OAuth 凭据每 5 分钟扫描一次，并在 token 生命周期 80% 或配置的过期 skew 里更早的时间点刷新。
- 修复：同一个 OAuth 凭据的并发刷新现在会跨 HTTP client 路径合并，降低 refresh token rotation 竞争风险。
- 修复：后台 OAuth refresh maintainer 只会在代理监听成功后启动，并且 shutdown 时可以取消正在进行的刷新。
- 改进：Claude live OAuth smoke 脚本会从临时凭据副本中剥离 refresh token、限制 `Retry-After`、默认不保留敏感临时数据，并支持 emoji/非 BMP prompt 的 billing fingerprint。

#### 发布说明

- 不需要配置迁移。
- 现有 OAuth 凭据保持兼容。Clipal 启动后，会自动维护带有效 refresh token 的凭据。
- Claude live smoke 脚本现在会在剥离临时 refresh token 后拒绝使用已过期 access token；请先重新授权或刷新源凭据，再运行 smoke。
