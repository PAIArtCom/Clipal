## v0.19.1

### English

#### Highlights

- Synced Claude OAuth wrapping with Claude Code `2.1.196`.
- Preserved valid user-supplied `Anthropic-Beta` values while still appending the OAuth wrapping betas Clipal needs for Claude-compatible requests.
- Updated Codex setup guidance for the latest config precedence behavior.

#### What Changed

- Improved: Claude OAuth wrapping now targets Claude Code `2.1.196`, including the current OAuth user agent, usage user agent, smoke-test payload fingerprints, and billing fingerprint.
- Fixed: Claude OAuth request normalization now merges valid inbound `Anthropic-Beta` values instead of replacing them wholesale.
- Hardened: Non-date-shaped spoofed beta tokens from official-looking Claude Code requests continue to be dropped before Clipal adds the required OAuth wrapping beta values.
- Updated: Codex takeover warning and client setup docs now state that project-local `.codex/config.toml` ignores model provider settings, so Clipal provider settings should remain in user-level `~/.codex/config.toml`.

#### Release Notes

- No config migration is required.
- Existing OAuth credentials remain compatible.
- This is a compatibility patch for Claude Code and Codex setup guidance after `v0.19.0`.

### 中文

#### 亮点

- 将 Claude OAuth 包装同步到 Claude Code `2.1.196`。
- 在追加 Claude 兼容请求所需 OAuth 包装 beta 的同时，保留有效的用户自定义 `Anthropic-Beta`。

#### 变更内容

- 改进：Claude OAuth 包装对齐 Claude Code `2.1.196`，包括当前 OAuth user agent、usage user agent、smoke-test payload fingerprint 和 billing fingerprint。

- 加固：对于伪装成官方 Claude Code 请求但不符合日期格式的 beta token，Clipal 仍会在追加 OAuth 包装所需 beta 前丢弃。
- 更新：Codex 接管提示和客户端接入文档现在说明 project-local `.codex/config.toml` 会忽略 model provider 设置，Clipal provider 应保留在用户级 `~/.codex/config.toml`。

#### 发布说明

- 不需要配置迁移。
- 现有 OAuth 凭据保持兼容。
- 这是 `v0.19.0` 之后面向 Claude Code 与 Codex 设置说明的兼容性补丁。
