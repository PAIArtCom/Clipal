## v0.20.0

### English

#### Highlights

- Replaced new Gemini CLI OAuth authorization with Antigravity-backed Google OAuth.
- Added Antigravity OAuth request wrapping for Gemini-compatible model listing, `generateContent`, `streamGenerateContent`, and `countTokens`.
- Expanded Google AI Studio and Vertex Gemini REST routing while preserving OpenAI-compatible resource routing priority.

#### What Changed

- Added: Gemini client OAuth authorization now offers `oauth_provider: antigravity`; legacy `oauth_provider: gemini` configs still load for compatibility, but new Gemini CLI OAuth authorization is no longer offered.
- Added: Antigravity OAuth uses the Cloud Code daily endpoint with Antigravity client metadata, version/user-agent detection, project metadata, and controlled upstream OAuth context.
- Added: Antigravity model listing normalizes upstream model data into Gemini API-compatible `models[]` responses.
- Added: Antigravity generation and streaming wrap Gemini-compatible request bodies into Antigravity internal envelopes while preserving official request fields such as `request_id`, `user_prompt_id`, `request_type`, and `enabled_credit_types`.
- Added: Antigravity `countTokens` support preserves Gemini request fields such as `contents`, `generationConfig`, `systemInstruction`, `tools`, and `cachedContent`.
- Improved: Google AI Studio / Gemini API-key routing now covers Gemini REST resources including embeddings, Imagen/Veo `predict*`, files, cached contents, tuned models, interactions, batches, operations, file search stores, generated files, corpora, auth tokens, agents, and webhooks.
- Improved: Vertex Gemini REST routing now covers publisher model methods and project/location operations paths.
- Fixed: Ambiguous `/clipal/files`, `/clipal/batches`, `/clipal/v1/files`, and `/clipal/v1/batches` paths remain OpenAI-compatible by default; Gemini keeps explicit Google-style resource methods such as `/files:register` and `/batches/{batch}:cancel`.
- Updated: README, setup docs, config reference, and the Gemini example config now document Antigravity OAuth, Google AI Studio, Vertex Gemini REST, and the Gemini Live API WebSocket limitation.

#### Release Notes

- No YAML migration is required for existing providers.
- Existing legacy `oauth_provider: gemini` providers remain loadable, but new Google personal OAuth providers should be authorized as Antigravity.
- Google AI Studio API-key providers should continue to use `https://generativelanguage.googleapis.com`.
- Vertex Gemini REST providers should use a regional `aiplatform.googleapis.com` base URL and an OAuth/service-account access token in `api_key` or `api_keys`; Clipal forwards it as `Authorization: Bearer ...` for Vertex hosts.
- Gemini Live API WebSocket proxying remains out of scope for this release.
- Live verification before release covered Antigravity model listing, text generation, token counting, SSE streaming, `gemini-3.1-flash-image` reachability, OpenAI-compatible resource routing priority, and Vertex operations routing.

### 中文

#### 亮点

- 将新的 Gemini CLI OAuth 授权入口切换为 Antigravity-backed Google OAuth。
- 增加 Antigravity OAuth 请求包装，支持 Gemini-compatible 模型列表、`generateContent`、`streamGenerateContent` 和 `countTokens`。
- 补齐 Google AI Studio 与 Vertex Gemini REST 路由，同时保持 OpenAI 兼容资源端点优先。

#### 变更内容

- 新增：Gemini 客户端 OAuth 授权现在提供 `oauth_provider: antigravity`；历史 `oauth_provider: gemini` 配置仍可加载兼容，但新的 Gemini CLI OAuth 授权入口不再提供。
- 新增：Antigravity OAuth 使用 Cloud Code daily endpoint，并带上 Antigravity client metadata、版本/user-agent 检测、project metadata，以及由 Clipal 生成的上游 OAuth 上下文。
- 新增：Antigravity 模型列表会归一化为 Gemini API-compatible 的 `models[]` 响应。
- 新增：Antigravity 生成与流式请求会把 Gemini-compatible 请求体包装为 Antigravity internal envelope，同时保留官方请求字段，如 `request_id`、`user_prompt_id`、`request_type`、`enabled_credit_types`。
- 新增：Antigravity `countTokens` 支持保留 Gemini 请求字段，如 `contents`、`generationConfig`、`systemInstruction`、`tools`、`cachedContent`。
- 改进：Google AI Studio / Gemini API-key 路由现在覆盖更多 Gemini REST 资源，包括 embeddings、Imagen/Veo `predict*`、files、cached contents、tuned models、interactions、batches、operations、file search stores、generated files、corpora、auth tokens、agents 和 webhooks。
- 改进：Vertex Gemini REST 路由现在覆盖 publisher model methods，以及 project/location operations 路径。
- 修复：`/clipal/files`、`/clipal/batches`、`/clipal/v1/files`、`/clipal/v1/batches` 这类有歧义路径默认保持 OpenAI 兼容；Gemini 保留明确的 Google 风格资源方法，如 `/files:register`、`/batches/{batch}:cancel`。
- 更新：README、接入文档、配置参考和 Gemini 示例配置现在说明 Antigravity OAuth、Google AI Studio、Vertex Gemini REST，以及 Gemini Live API WebSocket 暂不支持。

#### 发布说明

- 现有 provider 不需要 YAML 迁移。
- 历史 `oauth_provider: gemini` provider 仍可加载，但新的 Google 个人 OAuth provider 应使用 Antigravity 授权。
- Google AI Studio API-key provider 继续使用 `https://generativelanguage.googleapis.com`。
- Vertex Gemini REST provider 应使用区域 `aiplatform.googleapis.com` base URL，并把 OAuth / service-account access token 放在 `api_key` 或 `api_keys`；Clipal 会对 Vertex host 以 `Authorization: Bearer ...` 转发。
- Gemini Live API WebSocket 代理仍不在本次发布范围内。
- 发布前 live 验证覆盖了 Antigravity 模型列表、文本生成、token 计数、SSE 流式、`gemini-3.1-flash-image` 可达性、OpenAI 兼容资源端点优先级，以及 Vertex operations 路由。
