## v0.22.1

### English

#### Highlights

- Turned **Preview Import** into a real import-review dialog before any data is written.

#### What Changed

- The Web UI now reviews selected files, detected source, import mode, planned configuration/provider/credential/usage effects, and warnings in the existing modal design before Apply is available.
- Preview plans now bind to the current configuration, OAuth credentials, and usage state. If that state or the selected import data changes, Apply returns a conflict and requires a new preview.
- Canceling the review discards the plan; Apply happens only from the reviewed dialog and no longer opens a second browser confirmation.

#### Release Notes

- No backup schema or YAML migration is required.
- Existing `clipal.data/v1` backup files remain compatible.

---

### 中文

#### 亮点

- 将“预览导入”改为真正的数据写入前导入审查对话框。

#### 变更内容

- Web UI 现在会在执行前，使用既有 modal 设计审查所选文件、识别结果、导入模式、计划中的配置/Provider/凭据/用量影响以及警告。
- 预览计划现在绑定当前配置、OAuth 凭据和用量状态。若这些状态或所选导入数据发生变化，执行会返回冲突并要求重新预览。
- 取消审查会丢弃计划；只能从已审查的对话框执行导入，且不再出现第二个浏览器确认框。

#### 发布说明

- 不需要备份 schema 或 YAML 迁移。
- 现有 `clipal.data/v1` 备份文件保持兼容。
