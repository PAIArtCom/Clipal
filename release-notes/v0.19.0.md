## v0.19.0

### English

#### Highlights

- Updated Claude and Codex OAuth upstream wrapping to the current client fingerprints.
- Added OAuth wrapping for Claude and Codex so ordinary clients use Agent SDK-compatible envelopes by default.
- Updated Codex OAuth request metadata handling to preserve Codex CLI context while removing header-driven body pass-through.

#### What Changed

- Improved: Claude OAuth wrapping now targets Claude Code `2.1.195`, including the current OAuth user agent and billing fingerprint.
- Improved: Codex OAuth wrapping now targets Codex `0.142.4`, including the current Codex CLI `User-Agent`, `Originator`, and `Version` headers.
- Added: Claude OAuth requests use a lighter Agent SDK-compatible envelope by default, with current transport headers and billing fingerprints generated by Clipal.
- Added: Codex OAuth Responses requests are normalized into the lightweight Agent SDK-compatible shape, with required transport fields handled by Clipal.
- Fixed: Codex OAuth no longer uses caller-controlled `User-Agent` / `Originator` headers to select body-normalization pass-through.
- Fixed: Codex OAuth now preserves Codex CLI session, thread, parent-thread, turn-state, and turn-metadata context when present, and only fills missing context fields locally.
- Improved: Explicit client request fields are preserved when supported, including `tools`, Claude `thinking` / `context_management` / `output_config`, and Codex `reasoning` / `tool_choice` / `parallel_tool_calls`.
- Updated: README, docs, and landing page copy now describe the Agent SDK default mode, required transport handling, and explicit-field precedence.

#### Release Notes

- No config migration is required.
- Existing OAuth credentials remain compatible.
- The new wrapping behavior is covered by local tests; live OAuth smoke testing is still recommended before publishing a production release.

### 中文

#### 亮点

- 将 Claude 和 Codex OAuth 上游包装更新到当前客户端指纹。
- 为 Claude 和 Codex 增加 OAuth 包装，普通客户端默认使用 Agent SDK 兼容 envelope。
- 更新 Codex OAuth 请求 metadata 处理：保留 Codex CLI 上下文，同时移除由 header 驱动的 body 透传。

#### 变更内容

- 改进：Claude OAuth 包装对齐 Claude Code `2.1.195`，包括当前 OAuth user agent 和 billing fingerprint。
- 改进：Codex OAuth 包装对齐 Codex `0.142.4`，包括当前 Codex CLI `User-Agent`、`Originator` 和 `Version` header。
- 新增：Claude OAuth 请求默认使用更轻的 Agent SDK 兼容 envelope，并由 Clipal 生成当前传输 header 和 billing fingerprint。
- 新增：Codex OAuth Responses 请求会归一化为轻量 Agent SDK 兼容形态，并由 Clipal 处理必要传输字段。
- 修复：Codex OAuth 不再使用调用方可控的 `User-Agent` / `Originator` header 来选择 body-normalization 透传。
- 修复：Codex OAuth 会保留 Codex CLI 传入的 session、thread、parent-thread、turn-state 和 turn-metadata 上下文，只在缺失时由本地补齐。
- 改进：在目标模型支持时保留客户端显式字段，包括 `tools`、Claude `thinking` / `context_management` / `output_config`，以及 Codex `reasoning` / `tool_choice` / `parallel_tool_calls`。
- 更新：README、文档和 landing page 文案现在说明 Agent SDK 默认模式、必要传输字段处理，以及显式字段优先规则。

#### 发布说明

- 不需要配置迁移。
- 现有 OAuth 凭据保持兼容。
- 新包装行为已由本地测试覆盖；正式发布前仍建议运行 live OAuth smoke。
