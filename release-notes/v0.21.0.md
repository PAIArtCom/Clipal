## v0.21.0

### English

#### Highlights

- Fixed Antigravity OAuth plan detection so Google AI Pro accounts show their paid tier instead of being flattened to Free.
- Added Antigravity quota retrieval and UI metadata based on Cloud Code `retrieveUserQuota` and `retrieveUserQuotaSummary`.
- Improved provider-aware Gemini usage handling so Antigravity cooldowns use Antigravity quota data.

#### What Changed

- Fixed: Antigravity credential metadata now preserves `paidTier`, `currentTier`, and default allowed tier information separately.
- Fixed: plan display now prioritizes paid tier metadata for Antigravity accounts while still using current/default tiers for onboarding.
- Added: web OAuth metadata can return Antigravity quota buckets and quota summary groups.
- Added: Antigravity usage cache entries are keyed by OAuth provider and credential ref, preventing Gemini and Antigravity usage data from colliding.
- Improved: Antigravity refresh clears provider-specific usage cache data so quota changes appear after refresh.
- Improved: legacy Antigravity credentials with only historical `tier_id` metadata are refreshed into the newer paid/current/default tier shape.
- Fixed: volatile Antigravity tier metadata from an older credential is not merged into refreshed credentials, preventing stale Pro or Free labels from surviving account changes.
- Fixed: proxy cooldown checks now request usage through the normalized provider path, so Antigravity providers no longer fall back to Gemini quota retrieval.
- Added: focused tests for Antigravity tier extraction, credential metadata refresh, quota parsing, provider-aware cooldown behavior, and web metadata responses.

#### Release Notes

- No YAML migration is required.
- Existing Antigravity OAuth credentials continue to load; refreshing or reauthorizing will populate the richer tier and quota metadata.
- Google AI Pro display is based on `paidTier` when available. `currentTier` and allowed default tier remain available as separate metadata because they can represent runtime/default routing rather than paid subscription status.
- Gemini Live API WebSocket proxying remains out of scope for this release.

### 中文

#### 亮点

- 修复 Antigravity OAuth 计划识别，Google AI Pro 账号会显示付费 tier，不再被压平成 Free。
- 增加基于 Cloud Code `retrieveUserQuota` 和 `retrieveUserQuotaSummary` 的 Antigravity 额度获取与 UI 元数据。
- 改进 provider-aware Gemini usage 处理，Antigravity 冷却逻辑使用 Antigravity 自己的额度数据。

#### 变更内容

- 修复：Antigravity 凭据元数据现在会分别保存 `paidTier`、`currentTier` 和默认 allowed tier。
- 修复：Antigravity 账号的计划展示优先使用 paid tier，同时 onboard 仍使用 current/default tier。
- 新增：Web OAuth metadata 可以返回 Antigravity quota buckets 和 quota summary groups。
- 新增：Antigravity usage cache 现在按 OAuth provider 和 credential ref 分组，避免 Gemini 与 Antigravity 额度数据互相污染。
- 改进：Antigravity refresh 会清理对应 provider 的 usage cache，刷新后可以看到新的额度变化。
- 改进：只有历史 `tier_id` 元数据的旧 Antigravity credential 会被刷新成新的 paid/current/default tier 结构。
- 修复：刷新 credential 时不会把旧凭据里的易变 Antigravity tier 元数据合并回去，避免账号变化后残留过期的 Pro 或 Free 标签。
- 修复：代理冷却检查现在通过规范化 provider 路径获取 usage，Antigravity provider 不再回退到 Gemini quota retrieval。
- 新增：覆盖 Antigravity tier 提取、credential metadata refresh、quota 解析、provider-aware cooldown 和 web metadata response 的测试。

#### 发布说明

- 现有 YAML 不需要迁移。
- 现有 Antigravity OAuth credential 仍可加载；刷新或重新授权后会写入更完整的 tier 和 quota 元数据。
- Google AI Pro 展示以 `paidTier` 为准。`currentTier` 与默认 allowed tier 会作为独立元数据保留，因为它们可能表示运行时/default routing，而不是付费订阅状态。
- Gemini Live API WebSocket 代理仍不在本次发布范围内。
