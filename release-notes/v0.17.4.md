## v0.17.4

### English

#### Highlights

- Fixed Codex OAuth non-streaming Responses output for current `gpt-5.5` traffic.

#### What Changed

- Fixed: Codex OAuth non-streaming `/v1/responses` requests now preserve completed response metadata while filling an empty final `output` array from earlier `response.output_item.done` or `response.output_text.done` stream events.
- Improved: Added a regression test that mirrors the live Codex SSE sequence where `response.completed.response.output` is empty even though completed output items were already emitted.

#### Thanks

- Thanks to the live Codex OAuth verification that isolated this as a response synthesis bug rather than an account quota issue.
