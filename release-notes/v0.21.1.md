## v0.21.1

### English

#### Highlights

- Fixed Claude Code OAuth authorization to use the Claude app authorization endpoint.
- Improved Claude Code CLI takeover so Clipal can launch Claude Code without requiring a separate Claude client login.

#### What Changed

- Fixed: Claude OAuth authorization now starts from `https://claude.com/cai/oauth/authorize`.
- Improved: Claude Code takeover inserts a placeholder `ANTHROPIC_API_KEY=clipal` only when neither `ANTHROPIC_API_KEY` nor `ANTHROPIC_AUTH_TOKEN` already exists.
- Improved: Claude Code takeover status now requires both the Clipal base URL and a Claude client auth environment variable.
- Updated: Web UI and setup docs describe the placeholder API key behavior and clarify that existing Claude auth values are preserved.

#### Release Notes

- No YAML migration is required.
- Existing Claude OAuth credentials continue to load; reauthorization will use the updated authorization endpoint.
- Existing `ANTHROPIC_API_KEY` and `ANTHROPIC_AUTH_TOKEN` values in Claude Code settings are not overwritten.

### 中文

#### 亮点

- 修复 Claude Code OAuth 授权入口，改用 Claude app 授权端点。
- 改进 Claude Code 一键接管，使 Clipal 可以在不要求额外 Claude 客户端登录的情况下启动 Claude Code。

#### 变更内容

- 修复：Claude OAuth 授权现在从 `https://claude.com/cai/oauth/authorize` 开始。
- 改进：Claude Code 接管仅在既没有 `ANTHROPIC_API_KEY` 也没有 `ANTHROPIC_AUTH_TOKEN` 时补入占位 `ANTHROPIC_API_KEY=clipal`。
- 改进：Claude Code 接管状态现在要求同时存在 Clipal base URL 和 Claude 客户端认证环境变量。
- 更新：Web UI 与接入文档说明占位 API key 行为，并明确保留已有 Claude 认证值。

#### 发布说明

- 现有 YAML 不需要迁移。
- 现有 Claude OAuth credential 仍可加载；重新授权会使用更新后的授权端点。
- Claude Code settings 中已有的 `ANTHROPIC_API_KEY` 和 `ANTHROPIC_AUTH_TOKEN` 不会被覆盖。
