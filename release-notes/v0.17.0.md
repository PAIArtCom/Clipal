## v0.17.0

### English

#### Highlights

- Added usage-aware OAuth cooldowns so exhausted Codex, Gemini, and Claude OAuth accounts back off against provider-reported reset windows instead of retrying blindly.
- Added provider-level spend telemetry with daily and 7-day USD rollups, plus token breakdown tracking for reasoning and thoughts.
- Improved provider usage cards so official token semantics stay visually aligned: reasoning is shown under output, thoughts are shown under total, and hover text explains the exact totals.

#### What Changed

- Added: dynamic OAuth cooldown handling based on provider usage and retry-after signals for Codex, Gemini, and Claude OAuth accounts.
- Added: cost extraction and fallback pricing for OAuth-backed usage so provider cards can accumulate USD spend by day and week.
- Added: telemetry persistence for reasoning tokens, thoughts tokens, total cost, and daily cost buckets.
- Added: Web UI spend display for provider cards, including compact formatting with exact hover values.
- Improved: usage extraction now preserves reasoning/thought token semantics across JSON and SSE responses, including plain-text SSE responses that should be treated as event streams.
- Improved: provider usage cards now attach reasoning and thoughts to their parent metrics instead of rendering them as misleading peer totals.

#### Thanks

- Thanks to live testing and review feedback for pushing the usage, cooldown, and pricing semantics into release-ready shape.

### 中文

#### 亮点

- 新增基于 usage 的 OAuth 智能冷却，Codex、Gemini、Claude OAuth 账号会按 provider 返回的重置窗口退避，不再盲目重试。
- 新增 provider 级消费遥测，支持按美元累计日维度和近 7 天维度消费，同时记录 reasoning / thoughts token 拆分。
- 改进 provider usage 卡片展示，确保 token 语义与官方定义对齐：reasoning 挂在 output 下，thoughts 挂在 total 下，hover 会解释精确总量关系。

#### 变更内容

- 新增：针对 Codex、Gemini、Claude OAuth 账号，基于 provider usage 与 retry-after 信号的动态冷却处理。
- 新增：OAuth usage 的消费提取与 fallback 定价逻辑，使 provider 卡片可以累计按日与近 7 天美元消费。
- 新增：遥测持久化字段，记录 reasoning tokens、thoughts tokens、total cost 与 daily cost buckets。
- 新增：Web UI provider 卡片消费展示，包含紧凑金额格式与精确 hover 数值。
- 改进：usage 提取在 JSON 与 SSE 响应中都保留 reasoning / thought token 语义，包括应按 event stream 处理的 plain-text SSE 响应。
- 改进：provider usage 卡片将 reasoning 与 thoughts 挂到各自父指标下，不再作为容易误导的同级总量独立展示。

#### 致谢

- 感谢 live 测试与 review 反馈，帮助把 usage、cooldown 和 pricing 语义收敛到可发布状态。
