## v0.22.0

### English

#### Highlights

- Added complete Clipal data backup and restore through one canonical `clipal.data/v1` format.
- Added `clipal export` and `clipal import`, plus dedicated **Export Full Backup** and **Import Data** panels in the Web UI.
- Added transactional import protection for configuration, OAuth credentials, and usage history.

#### What Changed

- Full backups include global and per-client configuration, providers and API keys, OAuth access and refresh tokens, and usage data.
- Imports can preview a plan before applying it, then verify that the reviewed data has not changed. Native Clipal backups support replace or merge; supported CLIProxyAPI, sub2api, and Codex `auth.json` exports import credentials in merge mode.
- Imports coordinate configuration reloads, telemetry, and OAuth refreshes so a concurrent refresh cannot overwrite imported credentials or revive a removed account. Failed imports restore their prior configuration, credentials, and usage data; if credential rollback itself cannot complete, Clipal preserves the original credential backup path for manual recovery.
- The Web UI keeps export and import as separate panels. Export opens a save-location dialog when the browser supports it, with a browser-download fallback that identifies the downloaded file.
- The data import file picker now uses the same styled control as Provider OAuth file import.

#### Release Notes

- No YAML migration is required. Existing configuration and OAuth credentials are unchanged until an import is applied.
- Native backup schema version 1 is the new Clipal backup contract; the removed legacy configuration-export endpoint is not part of this release contract.
- Complete backups contain OAuth tokens and API keys. Store backup files with the same care as the source configuration directory.

### 中文

#### 亮点

- 新增完整的 Clipal 数据备份与恢复能力，统一采用 `clipal.data/v1` 标准格式。
- 新增 `clipal export`、`clipal import`，以及 Web UI 中独立的“导出完整备份”和“导入数据”面板。
- 导入配置、OAuth 凭据和用量历史时新增事务性保护。

#### 变更内容

- 完整备份包含全局与各客户端配置、Provider 和 API Key、OAuth access/refresh token，以及用量数据。
- 导入会先预览计划，再在执行时确认已审查的数据没有变化。原生 Clipal 备份支持替换或合并；受支持的 CLIProxyAPI、sub2api 和 Codex `auth.json` 导出会以合并模式导入凭据。
- 导入会协调配置热重载、遥测与 OAuth 刷新，避免并发刷新覆盖已导入凭据或重新写回已删除账号。导入失败时会恢复原有配置、凭据和用量；若凭据回滚本身失败，Clipal 会保留原始凭据备份路径，便于手动恢复。
- Web UI 将导出与导入拆分为独立面板。支持的浏览器会在导出时打开保存位置对话框；不支持时会回退为浏览器下载并提示文件名。
- 数据导入的文件选择器现在与 Provider OAuth 文件导入使用相同的样式。

#### 发布说明

- 不需要 YAML 迁移。除非执行导入，现有配置和 OAuth 凭据不会改变。
- 原生备份 schema version 1 是新的 Clipal 备份契约；已移除的旧配置导出端点不属于本次发布契约。
- 完整备份包含 OAuth token 和 API Key，请像保护源配置目录一样妥善保管备份文件。
